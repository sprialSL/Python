# 通知撰写技能

## 使用场景
当用户要求撰写、修改或润色通知类文档时使用此技能。

## 规则要求
1. **标题格式**：通知不能以"通知"二字开头，必须冠以"XX部"的前缀，例如"采购部通知"、"宣传部通知"、"销售部通知"等。
2. **部门处理**：
   - 如果用户明确告知所在部门，使用用户指定的部门名称，例如"销售部通知"
   - 如果用户没有告知所在部门，使用"XX部"代替，例如"XX部通知"
3. **通知内容**：根据用户提供的主题和内容要求，撰写符合规范的通知文档。

## 输出格式
```
[部门名称]通知

[通知正文]
```

## 示例
- 用户说："帮我写一个五一节放假的通知" → 输出"XX部通知"
- 用户说："我是销售部的，帮我写一个五一节放假的通知" → 输出"销售部通知"
